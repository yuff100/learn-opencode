## 学完你能做什么

- 用 AI 构思故事大纲和世界观

- 用 AI 辅助发展情节和设计冲突
- 写出风格鲜明的对话

---

## 为什么用 OpenCode，而不是网页版 AI？

| 能力 | 网页版 AI | OpenCode |
|-----|----------|----------|
| 多文件管理 | ❌ 只能在一个对话里 | ✅ 大纲、角色卡、章节分开存储 |
| 人设一致性 | ❌ 长篇容易忘设定 | ✅ @角色卡 引用，AI 永远记得 |
| 世界观管理 | ❌ 每次重新描述 | ✅ AGENTS.md 固化设定 |
| 章节版本 | ❌ | ✅ Git 追踪每章修改历史 |
| 跨章节搜索 | ❌ | ✅ 全项目搜索角色/情节 |

## 你现在的困境

- 有故事想法，但不知道怎么发展成完整情节
- 角色写着写着就"人设崩塌"
- 对话太平淡，没有人物特色

## 什么时候用这一招

- 当你需要：创作一部有完整世界观和立体人物的小说
- 而且不想：写到一半发现情节逻辑有漏洞

## 核心思路

### 小说创作四部曲

| 阶段 | 内容 | AI 辅助方式 |
|-----|------|-----------|
| 构思 | 世界观、时间线、核心冲突 | Plan 模式讨论 |
| 角色 | 角色卡、关系图、人物弧光 | 用模板生成 |
| 情节 | 章节大纲、场景设计、伏笔 | 逐章规划 |
| 执行 | 写对话、描写场景 | Build 模式输出 |

## 跟我做

### 第 1 步：构思故事大纲

**为什么**  
先有骨架，再填血肉。

切换到 Plan 模式，输入：

```
我想写一个都市悬疑小说：
- 背景：某科技公司发生连环失踪案
- 主角：一个有社交恐惧的程序员
- 核心悬念：失踪者都在使用同一款 APP

请帮我设计：
1. 故事的三幕结构
2. 核心冲突和转折点
3. 结局的几种可能性
```

### 第 2 步：设计主要角色

**为什么**  
立体的角色让故事有生命力。

输入：

```
根据刚才的故事，帮我设计主角的角色卡，包括：
- 基本信息（姓名、年龄、职业）
- 外貌特征
- 性格特点（至少 3 个正面、2 个负面）
- 背景故事
- 核心动机
- 人物弧光（从什么状态变成什么状态）
```

::: tip 💡 保存角色卡
切换到 Build 模式，让 AI 把角色卡保存为 `characters/主角.md`，方便后续引用。
:::

### 第 3 步：设计章节大纲

<AdInArticle />

**为什么**  
有大纲才能保持情节连贯。

输入：

```
根据三幕结构，帮我设计前 5 章的大纲，每章包含：
- 章节标题
- 主要场景
- 本章目标（推动什么情节/揭示什么信息）
- 章末悬念
```

### 第 4 步：写第一章

**为什么**  
开始实际创作。

切换到 Build 模式：

```
@characters/主角.md 根据角色设定和第一章大纲，写出完整的第一章。
要求：
- 以第一人称视角
- 用一个引人入胜的场景开场
- 控制在 3000 字左右
- 保存为 chapters/第一章.md
```

### 第 5 步：写人物对话

**为什么**  
对话是展现人物性格的关键。

```
@characters/主角.md @characters/配角.md 写一段主角和配角的对话场景：
- 场景：两人第一次在公司走廊相遇
- 目的：暗示配角知道一些秘密
- 风格：主角结巴紧张，配角故作轻松
```

## 📋 魔法 Prompt

### 👤 角色卡生成
> 预期效果：生成包含性格、背景、动机的完整角色卡

```
## 角色
你是资深小说编辑，擅长塑造立体、有深度的角色形象。

## 任务
根据用户提供的基本信息，生成一份完整的角色卡。

## 输入信息
### 必填项
- 姓名：【角色名】
- 身份：【职业/身份】

### 选填项
- 年龄：【年龄?】
- 故事类型：【玄幻/都市/言情/悬疑?】
- 角色定位：【主角/反派/配角?】
- 特殊要求：【其他要求?】

## 输出规范
### 1. 外貌描写（100字左右）
> 重点描写最具辨识度的特征

### 2. 性格光谱
| 正面特质 | 负面特质 |
|---------|---------|
| 特质1 + 具体表现 | 缺点1 + 具体表现 |
| 特质2 + 具体表现 | 缺点2 + 具体表现 |
| 特质3 + 具体表现 | |

### 3. 背景故事（200字左右）
> 塑造人物动机的核心经历

### 4. 核心动机
- **外在目标**：他/她想要什么
- **内在需求**：他/她真正缺少什么

### 5. 致命弱点
> 是什么让他/她脆弱

### 6. 标志性特征
- 口头禅：
- 标志性动作：

### 7. 人物弧线
> 从【起点状态】→ 到【终点状态】

## 约束条件
- ✅ 外貌描写要有画面感
- ✅ 性格特点要有矛盾面（不是完人）
- ✅ 背景故事要能解释性格成因
- ❌ 避免脸谱化（全好或全坏）
- ❌ 避免与主流角色雷同
```

### 📖 章节大纲生成
> 预期效果：生成包含场景、冲突、悬念的章节大纲

```
## 角色
你是资深小说编辑，擅长结构设计和节奏把控。

## 任务
根据故事背景，为指定章节生成详细大纲。

## 输入信息
### 必填项
- 故事简介：【故事简介】
- 章节编号：第【N】章

### 选填项
- 前情提要：【上一章发生了什么?】
- 本章目标：【需要推进什么情节?】
- 涉及角色：@【角色卡文件?】

## 输出规范
### 章节标题
> 【有吸引力的标题】

### 开场场景
- 时间：
- 地点：
- 氛围：

### 本章主要事件（3-5 个）
1. 事件1：【描述】
2. 事件2：【描述】
3. ...

### 本章冲突点
- **外部冲突**：【描述】
- **内部冲突**：【描述】

### 角色情感变化
| 角色 | 开始状态 | 结束状态 |
|------|---------|---------|

### 章末悬念/钩子
> 【让读者想看下一章的设计】

### 与主线关联
> 本章如何推进主线剧情

## 约束条件
- ✅ 每章要有独立的小高潮
- ✅ 章末必须有悬念
- ✅ 与前后章节逻辑连贯
- ❌ 避免事件堆砌没有重点
- ❌ 避免与主线脱节
```

### 💬 对话润色
> 预期效果：润色对话，使其更符合人物性格、更有张力

```
## 角色
你是专业的小说对话编辑，擅长让对话生动、有层次、富有潜台词。

## 任务
润色用户提供的对话，使其更具文学性和戏剧张力。

## 输入信息
### 必填项
- 角色设定：@【角色卡文件】（或描述角色性格）
- 场景背景：【描述】
- 原始对话：

[粘贴对话]

### 选填项
- 对话目的：【要达成什么?】
- 情绪基调：【紧张/温馨/压抑?】

## 润色维度
1. **角色区分度**：每个角色说话方式不同
2. **潜台词**：表面说一套，内心另一套
3. **动作描写**：说话时的小动作、神态
4. **节奏控制**：有急有缓，有长有短
5. **情感层次**：情绪的起伏变化

## 输出规范
### 润色后对话
[润色后的完整对话]

### 润色说明
| 修改点 | 原文 | 修改后 | 理由 |
|--------|------|--------|------|

## 约束条件
- ✅ 保持原意不变
- ✅ 每个角色说话风格要有区分度
- ✅ 增加必要的动作/神态描写
- ❌ 避免过度书面化
- ❌ 避免对话过长（每轮3-5句为宜）
```

## 踩坑提醒

| 现象 | 原因 | 解决 |
|-----|-----|-----|
| 角色人设崩了 | 没有用角色卡 | 创建角色卡文件，每次写对话用 @ 引用 |
| 情节前后矛盾 | 没有整体大纲 | 先设计三幕结构和章节大纲 |
| 对话太书面 | 风格指令不明确 | 在提示词中加入风格要求 |
