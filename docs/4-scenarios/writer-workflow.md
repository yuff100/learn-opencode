## 核心思路

### AI 辅助写作的三阶段

| 阶段 | 模式 | 做什么 |
|-----|------|-------|
| 构思 | Plan | 明确主题、确定结构、列大纲 |
| 输出 | Build | 生成正式内容、创建文件 |
| 迭代 | Build | 修改、润色、扩写 |

### 为什么要分阶段

- **构思阶段不写文件**：避免产生半成品
- **想清楚再动手**：减少返工
- **迭代有依据**：基于大纲修改，不会跑偏

## 跟我做
<AdInArticle />

### 第 1 步：创建写作项目文件夹

**为什么**  
让所有写作内容集中管理。

```bash
mkdir ~/my-writing
cd ~/my-writing
opencode
```

### 第 2 步：切换到 Plan 模式做构思

**为什么**  
构思阶段只需要讨论，不需要创建文件。

按 <kbd>Tab</kbd> 确保状态栏显示 `Plan`。

输入：

```
我想写一篇关于"远程办公的利与弊"的文章，目标读者是职场新人，字数 800 字左右。请帮我：
1. 确定文章结构
2. 列出每个部分的要点
3. 建议一个吸引人的标题
```

**你应该看到**：AI 输出文章框架和大纲，但不会创建任何文件

### 第 3 步：讨论并完善大纲

**为什么**  
在动手写之前把方向定好。

根据 AI 的建议，继续讨论：

```
第二部分"远程办公的弊端"可以加入一个个人经历的例子吗？请修改大纲
```

### 第 4 步：切换到 Build 模式输出

**为什么**  
大纲确定后，开始生成正式内容。

按 <kbd>Tab</kbd> 切换到 `Build` 模式。

输入：

```
按照刚才讨论的大纲，写出完整的文章，保存为 远程办公的利与弊.md
```

**你应该看到**：AI 创建文件并写入完整文章

### 第 5 步：迭代修改

**为什么**  
初稿通常需要调整。

继续在 Build 模式下修改：

```
@远程办公的利与弊.md 开头太平淡了，用一个引人入胜的场景描写开场
```

**你应该看到**：AI 修改文件开头

## 📋 魔法 Prompt

### ✍️ 写作起步模板
> 预期效果：生成完整的文章框架和大纲，为后续写作奠定基础

```
## 角色
你是资深内容策划师，擅长结构化思考和内容规划。

## 任务
根据用户提供的主题，生成完整的文章框架和大纲。

## 输入信息
### 必填项
- 文章主题：【主题】
- 目标读者：【读者画像】

### 选填项
- 文章目的：【说服/科普/娱乐/教程?】（默认：科普）
- 字数要求：【字数】（默认：1500字）
- 语气风格：【专业严谨/轻松幽默/温暖治愈?】（默认：轻松幽默）

## 输出规范
1. **文章定位**
   - 核心观点（一句话）
   - 目标读者的痛点/需求
   - 读完能获得什么

2. **文章结构**
   | 部分 | 功能 | 预估字数 |
   |------|------|----------|
   | 开头 | 吸引注意 | xxx 字 |
   | 中间 | 核心内容 | xxx 字 |
   | 结尾 | 行动号召 | xxx 字 |

3. **每部分要点**
   - 开头：如何引入
   - 中间：核心论点 + 支撑素材
   - 结尾：总结 + 金句

4. **备选标题**（3 个）
   - 标题 1
   - 标题 2
   - 标题 3

## 约束条件
- ✅ 结构要清晰，每部分功能明确
- ✅ 要点要具体，不只是"讲干货"
- ✅ 考虑读者的阅读体验和注意力曲线
- ❌ 避免结构过于复杂
- ❌ 避免要点过于笼统
```

## 踩坑提醒

| 现象 | 原因 | 解决 |
|-----|-----|-----|
| AI 直接写了一堆内容但没条理 | 没有先做构思 | 先在 Plan 模式讨论大纲 |
| 修改后越改越乱 | 没有基于大纲迭代 | 回到大纲，明确要改什么 |
| 输出风格不统一 | 每次重新描述风格 | 在 AGENTS.md 固化风格偏好 |
