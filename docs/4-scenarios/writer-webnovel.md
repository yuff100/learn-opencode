## 核心思路

### 网文的核心要素

| 要素 | 说明 |
|-----|------|
| 黄金三章 | 前三章决定读者是否追读 |
| 爽点节奏 | 每 2000-3000 字一个小爽点 |
| 金手指 | 主角的外挂/系统/异能 |
| 升级节奏 | 从小怪到 Boss 的能力成长 |

### AI 辅助网文的优势

- **扩写提速**：大纲扩写成正文，日更无压力
- **人设一致**：用角色卡保持性格不崩
- **爽点设计**：AI 帮你设计打脸套路

## 跟我做

<AdInArticle />

### 第 1 步：设计金手指

**为什么**  
金手指是网文的核心吸引力。

切换到 Plan 模式：

```
我要写一本都市修仙网文，帮我设计一个有趣的金手指系统：
1. 系统名称和来源
2. 核心能力（如何帮助主角）
3. 成长机制（如何升级）
4. 限制条件（避免太过 OP）
5. 可能的有趣玩法
```

### 第 2 步：写黄金三章

**为什么**  
前三章决定读者是否收藏。

```
帮我设计黄金三章的内容：

第一章：危机开场 + 金手指获得
第二章：小试牛刀 + 建立目标  
第三章：第一次装逼打脸

每章包含：
- 开头悬念
- 核心事件
- 章末钩子
- 字数建议 3000 字/章
```

### 第 3 步：生成第一章正文

**为什么**  
把大纲扩写成完整章节。

切换到 Build 模式：

```
根据第一章大纲，写出完整的第一章，要求：
1. 用第一人称
2. 开场直接进入冲突（不要铺垫太多背景）
3. 金手指获得要有仪式感
4. 章末留下悬念
5. 字数 3000 字左右
6. 保存为 chapters/第001章.md
```

### 第 4 步：设计爽点节奏

**为什么**  
网文要持续给读者爽感。

```
帮我设计前 30 章的爽点节奏表：

格式：
| 章节 | 爽点类型 | 具体事件 | 爽度等级 |

爽点类型包括：
- 打脸（踩小人）
- 获得（得到宝物/能力）
- 突破（境界提升）
- 救美（英雄救美）
- 复仇（找回场子）
```

### 第 5 步：快速扩写日更

**为什么**  
用 AI 提速，保持更新频率。

创建角色设定文件后，用引用扩写：

```
@settings/金手指.md @characters/主角.md

根据以下大纲扩写第 5 章正文：
「主角利用系统能力，在拍卖会上识破一件赝品，打脸嘲笑他的富二代」

要求：
- 3000 字
- 打脸过程要详细描写围观者反应
- 对话占比 40%
- 保存为 chapters/第005章.md
```

## 📋 魔法 Prompt

### 🎬 黄金三章生成
> 预期效果：生成吸引读者的开篇三章大纲

```
## 角色
你是网文资深作者，擅长写让读者欲罢不能的开头。你深谙"黄金三章"法则。

## 任务
生成网文开头的"黄金三章"大纲，确保读者追读。

## 输入信息
### 必填项
- 网文类型：【玄幻/都市/系统流/重生】
- 主角姓名：【名字】
- 金手指：【主角的特殊能力/系统】

### 选填项
- 开篇场景：【穿越时刻/觉醒时刻/重生时刻?】
- 主角初始处境：【描述?】

## 输出规范
### 第一章：危机 + 金手指
- **开局冲突**：如何让读者 5 秒入戏
- **主角处境**：为什么值得同情/期待
- **金手指获得**：获得方式（要有仪式感）
- **章末悬念**：让读者必须看第二章

### 第二章：初试 + 目标
- **金手指小试**：展示金手指的实用性
- **小打脸**：第一次让人刮目相看
- **短期目标**：建立可实现的近期目标
- **章末爽点**：给读者一个小爽

### 第三章：爽点 + 钩子
- **大打脸**：完成短期目标，狠狠打脸
- **能力展示**：金手指更多可能性
- **更大危机/机遇**：暗示更宏大的世界
- **追读钩子**：让读者"必须看下一章"

## 约束条件
- ✅ 每章结尾必须有强烈的悬念或爽点
- ✅ 节奏要快，不要水字数
- ✅ 30 秒内让读者知道主角是谁、有什么、要干什么
- ❌ 避免大段世界观介绍
- ❌ 避免主角内心独白过长
- ❌ 避免金手指太 OP（要有成长空间）
```

### ⚡ 爽点扩写
> 预期效果：将大纲扩写成3000字正文，爽点到位

```
## 角色
你是网文资深作者，擅长写让读者欲罢不能的爽点场景。

## 任务
将大纲扩写成完整的网文章节正文。

## 输入信息
### 必填项
- 章节大纲：【简要大纲】
- 字数要求：【字数】

### 选填项
- 角色设定：@【角色卡文件?】
- 金手指设定：@【设定文件?】
- 视角：【第一人称/第三人称?】（默认：第一人称）

## 输出规范
### 正文结构
1. **开场**（10%）：承接上章，进入场景
2. **铺垫**（20%）：建立冲突/矛盾
3. **爽点**（40%）：打脸/装逼核心场景
4. **反应**（20%）：围观者震惊/敌人愤怒
5. **钩子**（10%）：章末悬念

### 爽点写作技巧
1. **打脸要详写**：围观者反应、敌人表情变化
2. **心理描写**：敌人从嘲笑到震惊的转变
3. **对话占比**：40% 左右，推进节奏
4. **金句点睛**：主角一句话让人记住

## 约束条件
- ✅ 打脸/装逼场景要详细描写
- ✅ 加入围观者反应和心理描写
- ✅ 对话要符合人物性格
- ✅ 章末必须留悬念
- ❌ 避免大段心理独白
- ❌ 避免节奏拖沓、水字数
- ❌ 避免对话过于书面化
```

## 踩坑提醒

| 现象 | 原因 | 解决 |
|-----|-----|-----|
| 开篇太平淡 | 没有直接进冲突 | 第一章开头就进入危机 |
| 金手指太 OP | 没设限制条件 | 设计成长机制和限制 |
| 人设前后不一致 | 长篇容易忘设定 | 用角色卡 + @ 引用 |
