# v1.1.38：实验性 Markdown 组件，多项修复

**发布日期**：2026年01月28日

---

## 一句话总结

这次更新添加了实验性的 OpenTUI Markdown 组件，并修复了多项工具和插件相关问题。

---

## 新功能：OpenTUI Markdown 组件

### 这次更新了什么？

在 Desktop 应用中新增了实验性的 OpenTUI Markdown 组件，提供更好的 Markdown 渲染体验。

### 如何启用？

这个功能是实验性的，需要通过配置启用。启用后可以获得更流畅的 Markdown 渲染效果。

---

## 问题修复：AGENTS.md 重复加载

### 之前的问题

在并行工具执行时，AGENTS.md 文件可能会被重复加载多次，影响性能。

### 现在的情况

修复了这个问题，AGENTS.md 只会被加载一次，提高了并行执行的效率。

---

## 问题修复：PartUpdated 未取消订阅

### 之前的问题

TaskTool 中的 PartUpdated 事件订阅有时没有被正确取消，导致内存泄漏。

### 现在的情况

感谢社区贡献者 @goniz 的修复，现在确保所有订阅都会被正确取消。

---

## 问题修复：CLI 标识显示

### 之前的问题

CLI 的品牌标识显示存在问题。

### 现在的情况

感谢社区贡献者 @mynameistito 的修复，CLI 品牌标识现在可以正确显示。

---

## 要不要升级？

| 情况 | 建议 |
|------|------|
| 使用 CLI | ✅ 推荐升级（修复标识显示） |
| 关注性能 | ✅ 推荐升级（修复内存泄漏） |
| 用得好好的 | 可以等下次 |

**升级命令**：

```bash
opencode upgrade
```
